Let's Lay This Out.

Positioning in CSS

February 27, 2016

Elements and The Box Model

Let's start with the basics. An element is an individual component of a web page, the basic building block of HTML. Most elements are written with a start tag and an end tag with content inbetween. They can be nested within each other to describe the structure of the page.

Elements populate the page in what's known as the CSS box model. Each element is like a box that consists of different layers.

When we position an element we are moving the entire box, not just the content.

I would also like to note that it is common to group elements together in a containing element, such as < div >. This allows you to format them easier by using less CSS code.

The Position of Elements

The position property has four common values: static, relative, absolute, or fixed. These allow you to edit the layout of your page.

Static Positioning

Static positioning is the normal flow of the web page. If you don't apply another position value to your element, it will default to static. Statically positioned elements will stack on top of one another. You do not need to define the position as static, however you could with the following code; 

             box {
              position:static;
             }
This layout is helpful for a simple, single column pages where everything stacks on top of each other.

Relative Positioning

Relative positioning stacks items nicely as well but gives you some freedom to move the elements around. This scheme moves an element in relation to where it would have been in the static flow. You can move it using the offset properties: left, right, top, or bottom. 

              box {
                position:relative;
                top:15px;
                left:15px;
              }
The example above takes the box element and moves it down and to the left of the where it would have been, but leaves it in the normal flow.

Absolute Positioning

Absolute positioning is a little tricky. When an element is given position:absolute it is placed in relation to the first parent element with a set position. If all the elements are normally/static positioned, it is taken out of the normal flow of the page and acts like the other elements aren't there. If you tell an element to go somewhere with absolute positioning, there it will stay.

If you have already set an elements position, the next element will be placed relative to the first element. If you haven't defined any positions and use position:absolute, the element will be placed in relative to the document body.

Just like in relative positioning, you use the offset properites to move an element around. 

            box
              position:absolute;
              top:15px;
              left:15px;
            }
This bit of code looks very similar to the block we used above, however it would result in a very different layout. Since the position is set as absolute, the top and bottom values that we set would take the box element and place it exactly 15px from the top and 15px from the left. There it will stay, regardless of what elements are in its path. You can also set a width property to avoid any overlapping elements.

Fixed Positioning

Fixed positioning is like gluing an element in place. It's a type of absolute positioning and shares all the same rules, except the element is positioned in relation to the browser window as opposed to other elements or the HTML. When a user scrolls down the page the element will be in the exact same place- it won't scroll with you. This is useful when you want to have an ever-present navigation bar on your page.

Go and Build!

Now that you know more about the position property, you can accomplish building a variety of layouts, limited only by your imagination!
