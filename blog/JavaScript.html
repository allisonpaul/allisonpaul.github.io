JavaScript and Ruby

Similarities and Differences.

March 26, 2016

Over the past few weeks I've gotten to know Ruby on a pretty good level. I wouldn't call us best friends, but I would say we are pretty friendly and spent quite a bit of time together. It is still a bit of an adjustment switching to JavaScript, but overall they really are not all that different. There are some changes in syntax and logic, but you can get the same tasks accomplished with similar steps in many cases.

Declaring Variables

In Ruby it is very easy to declare a variable. You just type the variable name equal to the variable value. 

name = value
In Javascript it is still easy, just a little longer. 
var name = value

Ruby Hashes v. Javascript Objects

While getting to know Ruby, I learned a bit about hashes. Hashes are key value pairs that store information in no particular order. 


              hash= {hash_key: hash_value, hash_key2: hash_value2}
In JavaScript, hashes are referred to as objects and written slightly differently. 

              var hash= {hash_key: hash_value, hash_key2: hash_value2,}
You can call upon the value using either bracket or dot notation. 

              hash[hash_key]
              hash.hash_key

Semicolons

Ah, such a simple symbol but such a tough thing to bang into my head. In Ruby semicolons are not used for statements, yet in JavaScript they are used after every statement but not after curly brackets. 


            var x=0; x++  //semicolon obligatory
            var x=0       //semicolon optional
                x++       //semicolon optional
The semicolon is only obligatory when you have two or more statements on the same line, and is optional to seperate statements on seperate lines. You SHOULD NOT put a semicolon after closing a curly brace. The only exception are when declaring a variable or function with empty values. 

            var object= {};
I highly suggesting out this page for a more complete guide to semicolons.

Loops

Looping in JavaScript is much more limited than in Ruby. While there are numerous options with Ruby, there are only for loops and while loops in JavaScript.

While loops are the more basic way to iterate through a set of data.They start with a condition and then execute a block of code while that condition is true. When the condition is no longer true, the code stops and the loop ends. Here is a while loop in Ruby. 


            while (x < 5)
              puts x
              x +=1
            end
Here is the same while loop in JavaScript. 

            while (x<5){
            console.log(x);
            x++;
            }
So while the syntax is different, the logic is the same.

For loops are formatted by setting a statement to be executed before beginning the loop, defining the condition for running the loop, and executing the code each time the loop runs. Here is a for loop in Ruby. 


          for n in 1..10
            if n <= 5
              puts n
            end
          end
The same for loop in Javascript 

          for (var x=1; x<=5; x++){
            console.log(x);
          }
Both blocks of code are taking variable x and mulptiply it by five until it reaches 5.

The Same Yet Different

In these small examples you can see that you can accomplish the same tasks but with very different languages. Ruby and Javascript are both object-oriented languages. They have similar value types (numbers, strings, booleans, functions). They store data in variables. They evaluate functions (okay, methods in Ruby), and they have similar loops, conditionals, and operators.
