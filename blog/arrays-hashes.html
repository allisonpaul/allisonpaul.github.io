Arrays and Hashes

What are they and when do I use them?

March 5, 2016

Arrays

An array is a data type object that contains ordered lists of other objects. These objects can be variables, strings, integers, expressions, or even other arrays.

Creating Arrays

You can create an array using array.new or with literal notation like my_array=[]. Remember, arrays in Ruby allow you to store any type of objects, so your array can contain any combination of objects.

Accessing Data in Arrays

The data that you set to the array goes inside of the square brackets. It's really easy to look up values within an array by using their index value. In Ruby, array indexes start from zero. 


            my_array=[1,2,3,4,5]
            my_array[0]

The last line of the about code is pointing to the number 1, since it is the first value of the array. You can also perform a reverse index lookup and start from the back of the array! In this case, the last item of the array will start with index value -1 and they will become smaller as you move to the left.

Hashes

Hashes are a collection of key value pairs. Similar to arrays, they have values assoiciated with indexes but here they are called keys. Every key is supplied with a value. Like in arrays, keys can be any object, the only stipulation is they most be unique to the hash.

Creating a Hash

You create a hash the same way as an array; either with liternation notation such as empty_hash={key => value} or with the constructor method empty_ hash=Hash.new. Note that in a hash you use curly braces instead of square brackets.

Keys and Values

To access the information in a hash, you call out the key in square brackets and will receive the value. 


                state_capitals= {
                    "Kentucky"=> "Frankfort",
                    "Ohio" => "Columbus",
                    "Texas" =>Austin"
                  }

In the above example we have a hash of state capitols in which the states are the keys, and the capitals are the values. To retrieve the capital of Kentucky you type state_capitals["Kentucky"]

Arrays vs Hashes

So, when do you use an array and when do you use a hash? Good question. We now know that both are collections used to store and retrieve data. An array is more like a list of the data that is in a certain order. In a hash the important thing isn't the order of the information but what the information is paired to. I saw an example comparing hashes and arrays to shopping lists, which I think is really helpful so I will do the same. An array is like the list itself. You have your list and you ask yourself, "Okay, whats the first thing on my list? Milk. Whats the second thing? Pasta." Great, so if your list was an array, list[0] would give you milk and list[1] would give you pasta.

Your hash would be more like the specific information for what is on your list. So you need milk, what kind of milk do you need? I'm a skim person myself. There's a whole lot of pasta, so what are you looking for exactly? Let's go with penne.list["milk"] would return "skim" and list["pasta"] gives you "penne". Starting to make a little more sense now?
